Code Execution in JavaScript

Memory Creation Phase: - JavaScript engine first allocates memory for all variables and functions. - Variables declared with var are assigned **undefined**. - Variables declared with let and const are allocated memory but remain **uninitialized** in the **Temporal Dead Zone (TDZ)** until the code execution phase. - Function declarations are stored as a complete reference, so they can be called even before their definition.

Execution Phase: - Code is executed line by line. - var variables are assigned their actual values during execution. - let and const are initialized when the code execution reaches their line. - Accessing `let` or `const` before initialization will throw a **ReferenceError**. - Functions are executed when they are called.
